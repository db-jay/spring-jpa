### 섹션 2. 프로젝트 환경설정
목표:
- 프로젝트 실행 환경 이해
- H2 데이터베이스 연결
- Spring Boot + JPA 설정 이해
- SQL 로그를 보며 동작 확인

현재 프로젝트에서 중요한 설정:

```properties
spring.datasource.url=jdbc:h2:tcp://localhost/~/datajpa
spring.datasource.username=sa
spring.datasource.password=
spring.datasource.driver-class-name=org.h2.Driver

spring.jpa.hibernate.ddl-auto=create
spring.jpa.properties.hibernate.format_sql=true

logging.level.org.hibernate.SQL=debug
```

학습 포인트:
- `spring.datasource.*` : DB 연결 정보
- `spring.jpa.hibernate.ddl-auto=create` : 실행 시 테이블 생성
- `spring.jpa.properties.hibernate.format_sql=true` : SQL 보기 좋게 출력
- `logging.level.org.hibernate.SQL=debug` : Hibernate SQL 로그 확인

주의:
- `ddl-auto=create`는 학습용으로는 편하지만 운영에서는 위험할 수 있다.
- JPA는 반드시 **실행 SQL 로그를 같이 보면서 공부**해야 한다.

## 4. 현재 프로젝트 기준 핵심 학습 포인트

### 4-1. 방금 적용한 설정의 의미
현재 메인 설정은 `application.yml`로 반영되어 있다.

즉, 지금 프로젝트는 아래를 학습하기 좋은 상태다.

- H2 DB 연결
- JPA 동작 확인
- Hibernate SQL 로그 확인
- 테이블 자동 생성 경험

---

### 4-2. SQL 로그를 보는 습관
JPA를 공부할 때는 반드시 SQL 로그를 같이 봐야 한다.

이유:
- 자바 객체 코드가 실제 SQL로 어떻게 번역되는지 알 수 있다.
- 성능 문제와 쿼리 문제를 빨리 발견할 수 있다.
- “JPA가 다 알아서 해준다”는 착각을 줄일 수 있다.

핵심 문장:
> JPA 학습은 객체 코드 + SQL 로그를 함께 봐야 한다.

---

### 4-3. 현재 프로젝트에서 확인된 상태
현재 설정 파일은 적용되었다.

다만 테스트 실행 시 `build.gradle`에서 테스트 의존성이 빠져 있어서 실패할 수 있다.
예를 들어 `spring-boot-starter-test`가 주석 처리되어 있으면 테스트 컴파일이 되지 않는다.

이건 설정 문제라기보다 **테스트 환경 구성이 덜 된 상태**다.

---

## 4-4. 오늘 진행한 내용 정리

오늘은 본격적인 JPA 학습을 시작하기 전에, **실행 환경과 테스트 환경을 먼저 안정화**했다.

진행한 내용:

1. **JUnit 테스트 환경 복구**
   - `spring-boot-starter-test` 의존성을 다시 활성화했다.
   - 테스트가 컴파일되지 않던 문제를 먼저 해결했다.

2. **`.gitignore` 추가**
   - macOS, IntelliJ, Gradle 빌드 결과물, 로컬 DB 파일 등을 ignore 하도록 정리했다.
   - 학습과 상관없는 파일이 git 상태를 어지럽히지 않도록 정리했다.

3. **메인 설정을 `application.yml`로 정리**
   - 기존 `application.properties`를 `application.yml`로 변환했다.
   - H2, JPA, 로그 설정을 한 곳에서 읽기 쉽게 정리했다.

4. **테스트 전용 DB 설정 분리**
   - `src/test/resources/application.properties`를 추가했다.
   - 테스트는 `jdbc:h2:mem:testdb`를 사용하도록 분리했다.
   - 이유: 실행 중인 로컬 파일 DB와 테스트가 서로 충돌하지 않게 하기 위해서다.

5. **`Member` 엔티티 생성자 보완**
   - `new Member("memberA")` 형태로 테스트할 수 있도록 생성자를 추가했다.
   - 동시에 JPA 기본 생성자(`protected Member()`)도 유지했다.

6. **학습용 주석 추가**
   - 엔티티, 리포지토리, 설정 파일, 테스트 코드에
     “왜 이렇게 작성하는지”를 이해하기 위한 짧은 설명을 넣었다.

오늘 얻어야 하는 핵심 감각:

- 테스트 실행과 서버 실행은 다르다.
- JPA는 `persist()` 즉시 insert SQL을 보내지 않을 수 있다.
- 같은 트랜잭션 안에서는 1차 캐시 때문에 select가 안 보일 수 있다.
- 테스트 DB와 실행 DB는 분리하는 습관이 좋다.
